# ESP32 CYD — CAN Brightness Hunter

**Цель:** Найти CAN ID, управляющий яркостью подсветки приборки Honda Accord 8,
и управлять дополнительной LED-лентой.

**Плата:** ESP32-2432S028 (CYD / Cheap Yellow Display)

---

## 1. Подключение

| MCP2515 | ESP32 CYD |
|---------|-----------|
| VCC     | 5V        |
| GND     | GND       |
| SCK     | GPIO18    |
| MOSI    | GPIO23    |
| MISO    | GPIO19    |
| CS      | **GPIO32** |
| INT     | не подключать |

**CAN-шина авто → OBD2 разъём:**
- CAN_H → пин 6
- CAN_L → пин 14
- GND → пин 4
- +12V → пин 16 (через стабилизатор 5V для CYD)

**LED-лента:**
- GPIO26 → 100Ω → Gate IRLZ44N
- Drain → LED-лента (-)
- Source → GND
- Gate → 10kΩ → GND (pull-down)
- LED-лента (+) → +12V через предохранитель 1-2А

## 2. Логика работы (4 режима)

### Режим 1: IDLE — ожидание

Экран приветствия, инструкция и кнопка **START SCAN**.
```
┌──────────────────────────────┐
│      CAN HUNTER              │
│  MCP2515: OK (500K)          │
│                              │
│  1. Ключ в ON                │
│  2. START SCAN               │
│  3. Крути яркость            │
│  4. Выбери ID из списка      │
│                              │
│       ┌──────────┐           │
│       │START SCAN│           │
│       └──────────┘           │
└──────────────────────────────┘
```

### Режим 2: SCANNING — сбор данных (30 сек)

Собирает ВСЕ CAN ID на шине, считает:
- **count** — сколько раз ID пришёл
- **changes** — сколько раз менялись данные в нём

Чем активнее крутишь колёсико яркости — тем больше changes
у нужного ID.

```
┌──────────────────────────────┐
│      SCANNING...             │
│         23 s                 │
│  IDs found: 17              │
│  Packets: 1438               │
│ ████████████░░░░░░░░░░░░░░   │
│                              │
│  TOP ID:                     │
│   0x160  (214 pkts) <- самый │
│   0x130  (188 pkts)   активный│
│   0x100  ( 92 pkts)          │
└──────────────────────────────┘
```

### Режим 3: LIST — выбор ID

Список всех найденных ID, **отсортирован по changes** (самые меняющиеся — сверху).
Справа — количество изменений и полоска.

```
┌──────────────────────────────┐
│ FOUND IDs (sorted by chg) 1/3│ ← страница
│ ┌─────────────────────────┐  │
│ │0x160  rx:214  chg:52 ███│  │ ← тап сюда!
│ └─────────────────────────┘  │
│ ┌─────────────────────────┐  │
│ │0x374  rx:188  chg:3  ░░░│  │
│ └─────────────────────────┘  │
│ ┌─────────────────────────┐  │
│ │0x130  rx:92   chg:1  ░░░│  │
│ └─────────────────────────┘  │
│                              │
│   [<]  [>]     [SCAN AGAIN]  │
└──────────────────────────────┘
```

**Тап по ID → MONITOR** (начинает следить за ним)
**< >** — перелистывание страниц (если более 6 ID на страницу)
**SCAN AGAIN** — новый сбор

### Режим 4: MONITOR — слежение за ID

Показывает яркость из первого байта выбранного ID.
Большой индикатор, процент, ШИМ.

```
┌──────────────────────────────┐
│ MONITOR: 0x160          BACK │ ← кнопка назад
│     ┌──────────────────┐     │
│     │                  │     │
│     │      64%         │     │ ← крупно!
│     │                  │     │
│     │   ████████████   │     │ ← заливка по яркости
│     │   ████████████   │     │
│     │   ████████████   │     │
│     │   ████████████   │     │
│     └──────────────────┘     │
│ ID: 0x160  PWM: 163/255 RX:5│
│ Rotate brightness wheel      │
│ to see changes               │
└──────────────────────────────┘
```

**BACK** → возврат в список (можно выбрать другой ID)
LED-лента управляется в реальном времени (первый байт → ШИМ)

## 3. Поиск ID подсветки (алгоритм)

1. Включаешь зажигание (ON)
2. Жмёшь **START SCAN** на CYD
3. **30 секунд крутишь колёсико яркости** туда-сюда
4. После окончания скана открывается список ID
5. **Самые активные — наверху** (они меняли данные чаще всего)
6. Тапаешь по первому — если яркость на экране не меняется, жмёшь BACK → следующий
7. Когда нашёл — LED-лента будет повторять яркость синхронно с приборкой

## 4. Типовые проверенные ID

Из opendbc (commaai/openpilot) подтверждены только внешние световые ID:

| ID (hex) | Описание | Комментарий |
|----------|----------|-------------|
| 0x374 | STALK_STATUS (фары, дворники) | HEADLIGHTS_ON, AUTO_HEADLIGHTS |
| 0x37B | STALK_STATUS_2 (габариты, ближний/дальний) | PARK_LIGHTS, LOW_BEAMS |
| 0x35E | CAMERA_MESSAGES (автодальний) | HIGHBEAMS_ON |

**ID диммирования приборки Honda НЕ НАЙДЕНЫ в открытых источниках.**
Для Accord 8 (2008-2012) opendbp отсутствует — только сниффер поможет.

## 5. Пины CYD

Все пины, кроме GPIO4, **можно использовать** — плата одна, проекты разные.
Только GPIO4 (TFT_RST) реально занят — при дёргании дисплей сбрасывается.

| Пин | Назначение в этом проекте | Можно ли менять |
|-----|--------------------------|----------------|
| 18 | SPI SCK | Нет (дисплей + MCP2515) |
| 19 | SPI MISO | Нет |
| 23 | SPI MOSI | Нет |
| 4 | TFT_RST | НЕ ТРОГАТЬ — сброс дисплея |
| 5 | SD_CS | Да (слот SD не используется) |
| 22 | CAN_CS (альтернатива) | Да |
| 26 | LED_PWM | Да |
| 32 | CAN_CS (по умолчанию) | Да |

## 7. Безопасность

⚠️ **CAN-шина — это ABS, SRS (подушки), двигатель.**
- **Только чтение!** Скетч не отправляет сообщения на шину
- Отключай зажигание перед подключением/отключением модуля
- Питание LED-ленты через предохранитель
- Сначала тестируй на стенде
