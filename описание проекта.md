# Проект: CAN ID Hunter для ESP32 (CYD)

**Версия: 1.0**

## Описание

Прошивка превращает плату **ESP32-2432S028 (Cheap Yellow Display)** + MCP2515+TJA1050 в CAN-сниффер для поиска CAN ID на автомобильной шине.

Задача: найти CAN ID диммирования подсветки приборки на Honda Accord 8 (2008-2012), который отсутствует в открытых источниках (opendbc/commaai). Найдя ID — можно будет управлять подсветкой (управление LED вынесено в отдельный проект, этот — только сканер).

**Аппаратная часть:**
* ESP32-2432S028 (CYD) — основной контроллер + дисплей 320×240
* MCP2515 + TJA1050 — CAN-контроллер (SPI)

## Логика работы

### 4 режима:

**1. IDLE** — экран ожидания, инструкция, кнопка START SCAN

**2. SCANNING** — сбор данных с CAN-шины в течение 30 секунд:
- Собирает все ID на шине
- Считает **count** и **changes** (изменения данных)
- **Двухфазный приём:**
  1. SPI drain (MCP2515 → кольцевой FIFO на 512 сообщений, ~6.5 KB RAM)
  2. Неторопливая обработка FIFO (поиск/сравнение без SPI)
- MCP2515 имеет всего 3 аппаратных RX-буфера — FIFO исключает потерю пакетов
- Экран не обновляется во время скана — только таймер + прогресс-бар
- Кнопка STOP для досрочного завершения

**3. LIST** — найденные ID, сортировка по changes (убывание):
- 5 ID на страницу, пагинация < >
- Тап по ID → MONITOR
- SCAN AGAIN → возврат в IDLE

**4. MONITOR** — слежение за выбранным ID:
- Первый байт как значение яркости (0-100%)
- Большой индикатор с заливкой от низа
- BACK → список

### Поиск ID подсветки:
1. Включить зажигание (ON)
2. START SCAN на CYD
3. 30 секунд крутить колёсико яркости
4. В списке — сверху ID с наибольшим changes
5. Тапнуть первый → проверить реакцию шкалы
6. Если не тот → BACK → следующий

## Подключение MCP2515 к CYD

### Пайка (обязательно):

| Сигнал | GPIO CYD | Куда на CYD | Куда на MCP2515 |
|--------|----------|-------------|-----------------|
| **5V** | — | площадка S3 (сзади) | VCC |
| **GND** | — | площадка S1 (сзади) | GND |
| **SCK** | GPIO 18 | ножка ESP32 | SCK |
| **MOSI** | GPIO 23 | ножка ESP32 | SI |
| **MISO** | GPIO 19 | ножка ESP32 | SO |

### Без пайки (через разъём):

| Сигнал | GPIO CYD | Разъём | Пин |
|--------|----------|--------|-----|
| **CS** | GPIO 22 | **P3** (вид сзади — слева) | **пин 3** |
| CAN_H | — | MCP2515 клеммы | → OBD2 пин 6 |
| CAN_L | — | MCP2515 клеммы | → OBD2 пин 14 |

**Итого:** 5 точек пайки к ESP32 + CS втыкается в P3 (без пайки).

### Схема расположения на CYD:

```
                  CYD (вид сзади)
    ┌──────────────────────────────────────┐
    │                                      │
    │    ESP32:                            │
    │      · 18 (SCK) ──────────────────┐  │
    │      · 19 (MISO) ───────────────┐ │  │
    │      · 23 (MOSI) ────────────┐  │ │  │
    │                              │  │ │  │
    │   Площадки:                  │  │ │  │
    │   S1(GND) ────────────────┐ │  │ │  │
    │   S3(5V) ──────────────┐  │ │  │ │  │
    │                        │  │ │  │ │  │
    │   P3:                  │  │ │  │ │  │
    │   ┌──────────────────┐ │  │ │  │ │  │
    │   │ 1:GND 2:35       │ │  │ │  │ │  │
    │   │ 3:22 ← CS ←──┘  │ │  │ │  │ │  │
    │   │ 4:21             │ │  │ │  │ │  │
    │   └──────────────────┘ V  V V  V V  V
    │                         MCP2515 модуль
    │                         (5 проводов SPI +
    │                          1 провод CS в P3)
    └──────────────────────────────────────┘
```

### CAN шина → OBD2:
- CAN_H → пин 6 (разъём OBD2)
- CAN_L → пин 14 (разъём OBD2)
- GND → пин 4

## Безопасность
⚠️ **Только чтение!** Скетч не отправляет CAN-сообщения
CAN-шина — ABS, SRS (подушки), двигатель — будьте аккуратны
