**Версия: 2.0-alpha**

От простого CAN ID сниффера до полноценного **бортового диагностического центра**.
Один дешёвый ESP32-дисплей — безграничные возможности.

---

## Философия

**CAN-Мультитул** — проект, родившийся из практической задачи: найти ID диммирования подсветки
на Honda Accord 8. Начавшись как простой сниффер, проект перерос в универсальную платформу
для работы с CAN-шиной любого автомобиля.

В основе — **10 режимов** для разных задач: от сканирования шины до эмуляции CAN-устройств.
Всё это работает на CYD (ESP32-2432S028) с добавлением MCP2515 CAN-модуля — примерно 600₽
за всё железо.

---

## 10 режимов

### 1. Сканер CAN ID ✅
30-секундное сканирование шины. Собирает все ID, сортирует по частоте изменений (changes).
После сканирования — список с сортировкой. Тап по любому ID → просмотр значений.

### 2. Монитор значений ✅
Два подрежима:
- **RAW** — столбец 0–255 для первого байта, процент, сырые hex-байты
- **DECODE** — расшифровка данных по ID:
  - `0x17C` → RPM (data[2:3]), газ (%)
  - `0x309` → скорость (км/ч)
  - `0x324` → температура ОЖ и воздуха (°C)
  - `0x1A6` → уровень топлива (%), круиз-контроль
  - `0x191` → передача (P/R/N/D/2/1/L/S)
  - `0x1A4` → педаль тормоза (%)
  - `0x158` → скорость с точностью 0.01 км/ч
  - `0x13C` → положение дросселя (%)
  - `0x18E` → продольное/поперечное ускорение (G)
  - `0x294` → поворотники, одометр
  - `0x255` → скорости 4 колёс

### 3. Спидометр ✅
Весь экран — крупные цифры. Три подрежима:
- **SPD** — скорость (textSize 7)
- **RPM** — обороты (textSize 7)
- **BOTH** — скорость + RPM (textSize 5)

Кнопки режимов слева в подвале, MENU — красная кнопка справа.

### 4. Датчики двигателя ✅
Сетка 2×3: скорость, RPM, температура ОЖ, топливо, дроссель, напряжение бортовой сети.
Значения подставляются автоматически при появлении соответствующих ID на шине.

### 5. Поиск блоков (ECU SCAN) ⏳
UDS-сканирование — поиск всех электронных блоков на шине.
Карта модулей: ECU, ABS, SRS, BCM, EPS, климат, приборка.

### 6. CAN-логгер ⏳
Запись трафика на SD-карту в CSV. Колонки: время, ID, DLC, data[0–7].

### 7. OBD2 сканер ⏳
Стандартные OBD2 PID-запросы через 0x7DF → 0x7E8.
Не требует знания CAN ID.

### 8. Тест ID ⏳
Ручной ввод ID → мониторинг байт в реальном времени.

### 9. SLCAN мост ⏳
Режим USB-CAN адаптера для SavvyCAN/Python на ПК.

### 10. Эмулятор ⏳
Генерация тестовых CAN-пакетов для отладки.

## Меню и навигация

- **Главное меню**: сетка 2×2, 8 пунктов на 2 страницах
- **Индикаторы**: зелёный `+` — готово, красный `-` — в планах
- **Навигация**: `<` / `>` в подвале для переключения страниц
- **Подвал**: универсальная панель 30px с кнопками во всех режимах

```
┌──────────────────────────────────┐
│  ⚫ CAN-MULTITOOL        v2.0    │ ← Хедер с точкой CAN
├──────────────────────────────────┤
│  [+ SCAN ID]  [+ MONITOR]       │
│  [+ SPEEDO]   [+ SENSORS]       │ ← 2×2 сетка
│  [- ECU SCAN] [- LOGGER]        │
│  [- OBD2]     [- PROBE ID]      │
├──────────────────────────────────┤
│  [<]         MENU         [>]    │ ← Подвал
└──────────────────────────────────┘
```

## Железо

### Текущая конфигурация
| Компонент | Назначение |
|-----------|-----------|
| ESP32-2432S028 (CYD) | Контроллер + дисплей 320×240 с тачскрином |
| MCP2515 + TJA1050 | CAN-контроллер через SPI (CS=22) |

### Подключение (пайка)
| MCP2515 | CYD | Примечание |
|---------|-----|-----------|
| VCC (5V) | 5V S3 | Паять |
| GND | GND S1 | Паять |
| SCK | GPIO 18 | Паять |
| MOSI | GPIO 23 | Паять |
| MISO | GPIO 19 | Паять |
| CS | GPIO 22 | Провод на P3 pin 3 |
| CAN_H | OBD2 pin 6 | — |
| CAN_L | OBD2 pin 14 | — |

**Необходимые библиотеки Arduino:**
- `TFT_eSPI` (CYD variant)
- `MCP_CAN` v2.x (by coryjfowler)
- `Font7` (built-in, 7-segment RLE шрифт)
- `FreeSansBold` (built-in GFXFF, сглаженный anti-aliased шрифт)

### Апгрейды
1. **Второй MCP2515** — для B-CAN (125 кбит)
2. **RejsaCAN v6.x** — ESP32-C6, 2×CAN нативно, питание 12V от OBD2
3. **TWAI** — встроенный CAN ESP32 (без внешнего контроллера)

## Архитектура CAN (Honda Accord 8)

| Шина | Тип | Скорость | Что там |
|------|-----|----------|---------|
| **F-CAN** | High-Speed CAN | **500 кбит** | Двигатель, АКПП, ABS, EPS |
| **B-CAN** | High-Speed CAN | **125 кбит** | Свет, дворники, замки, диммер подсветки |

**Шлюз (приборная панель)**: передаёт выборочные сигналы:
- F→B: скорость (0x309), температура ОЖ
- B→F: зажигание, ремни, кнопки руля (как 0x296 SCM_BUTTONS)
- **НЕ передаёт**: диммер подсветки (0x341), поворотники, дворники, климат, замки
