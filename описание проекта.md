# ⚡ CAN-Мультитул — Бортовой диагностический центр на CYD

**Версия: 2.0 (в разработке)**

## 🌟 Великое Видение

**CAN-Мультитул** — это не просто сниффер. Это полноценный **бортовой диагностический центр** на ESP32+CYD, который превращает дешёвый дисплей за 500₽ в профессиональный инструмент для работы с CAN-шиной автомобиля.

От сканера CAN ID до графического дашборда, от поиска электронных блоков до логгера на SD-карту — всё в одном устройстве с сенсорным управлением.

Проект рождается из практической задачи (поиск ID диммирования подсветки на Honda Accord 8) и вырастает в **универсальную платформу** для любого авто.

## 🧩 Полный список режимов

### 🔍 Режим 1 — Сканер CAN ID *(готов на 100%)*
Классика — 30 секунд сканирования, сортировка по changes.
Перенесён из v1.2 без изменений.

### 📊 Режим 2 — Монитор значений
Выбираешь ID → видишь расшифрованные данные:
- `0x17C` → RPM: 2450 об/мин
- `0x309` → Скорость: 65 км/ч  
- `0x324` → Температура: 89°C
- `0x1A6` → Топливо: 42%
- `0x191` → Передача: D4
- Использует таблицу парсинга байт из HondaCAN

### 🏎️ Режим 3 — Спидометр / Тахометр
Весь экран — крупные цифры. Выбор: скорость / RPM / вместе.

### 🔬 Режим 4 — Датчики двигателя
Сетка 2×3 или 3×2:
| Скорость | RPM |
| Температура | Топливо |
| Напряжение | Газ |

### 🔎 Режим 5 — Поиск блоков (UDS)
Сканирует CAN-шину в поиске всех электронных блоков.
Посылает расширенный запрос на каждый ID → кто ответил — тот существует.
Показывает карту блоков: ECU, ABS, SRS, BCM, EPS, AC, CLUSTER...

### 📝 Режим 6 — CAN-логгер
Запись всего трафика на SD-карту.
Формат CSV: timestamp, ID, DLC, data[0-7].
Потом анализ на ПК через Python-дешифратор.

### 📡 Режим 7 — OBD2 сканер
Запрос через 0x7DF → 0x7E8.
Читает стандартные PID: скорость, RPM, температуру ОЖ, нагрузку, угол дросселя, lambda.
Не требует знания CAN ID — работает по OBD2 протоколу.

### 🧪 Режим 8 — Ручной тест ID
Ввод ID вручную → мониторинг байт в реальном времени.
Для реверс-инжиниринга неизвестных ID.

### 🌐 Режим 9 — SLCAN (SavvyCAN)
Режим USB-CAN интерфейса. Подключаешься к SavvyCAN на ПК — получаешь профессиональный анализ.

### 🔄 Режим 10 — CAN-эмулятор
Генерация тестовых CAN-пакетов.
Для отладки и проверки нового железа.

## 🛠️ Аппаратная часть

| Версия | Описание | Статус |
|--------|----------|--------|
| **Текущая** | CYD + MCP2515 + TJA1050 (1×CAN, 500 кбит) | ✅ Работает |
| **Апгрейд 1** | + второй MCP2515 (B-CAN 125 кбит) | ⏳ План |
| **Апгрейд 2** | RejsaCAN v6.x ESP32-C6 dual CAN | 📌 Цель |

### Текущая конфигурация (v1.2):
- ESP32-2432S028 (CYD) — контроллер + дисплей 320×240
- MCP2515 + TJA1050 — CAN-контроллер (SPI, CS=22)
- Подключение к OBD2 разъёму (пин 6 = CAN_H, пин 14 = CAN_L)

### Планы по железу:
1. Второй MCP2515 + TJA1050 для B-CAN (125 кбит)
2. Переход на TWAI (встроенный CAN ESP32) — без MCP2515
3. RejsaCAN v6.x — 2×CAN натив, питание 12V, автоотключение

## 🏆 5 фаз развития

### Фаза 1 — Фундамент ✅
- Сниффер v1.0 — базовый сканер + LIST
- Сниффер v1.1 — сортировка по changes
- Сниффер v1.2 — MONITOR layout, анти-мерцание
- Excel с таблицей HondaCAN ID
- GitHub-репозиторий

### Фаза 2 — Мультитул 🔧 *[СЕЙЧАС]*
- Главное меню с тач-кнопками
- Режим сканера (порт из v1.2)
- Режим монитора значений (decoder)
- Режим спидометр/тахометр
- Полное переименование в CAN-Мультитул

### Фаза 3 — Диагностика
- Поиск блоков (UDS)
- OBD2 сканер (PID)
- Кастомная база ID для разных авто
- Сохранение/загрузка профилей

### Фаза 4 — Профессионал
- CAN-логгер на SD
- SLCAN (SavvyCAN интерфейс)
- Поддержка B-CAN (второй MCP2515)
- Графики изменения байт

### Фаза 5 — Релиз
- Поддержка RejsaCAN v6.x (dual CAN)
- Нативная TWAI поддержка (без MCP2515)
- Графический дашборд (LVGL)
- BLE-вывод на телефон
